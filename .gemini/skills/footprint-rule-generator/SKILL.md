# Footprint Rule Generator

Этот скилл помогает создавать стандартизированные файлы правил в формате Markdown для проекта DIYLayout.

## Основной процесс создания правила

1. **Сбор данных**: Получите от пользователя или из даташита:
   - Название корпуса (например, SOT-23, ZIP-10).
   - Количество выводов.
   - Физические размеры и шаг (в мм или отверстиях 2.54 мм).
   - Схему нумерации выводов.
2. **Использование шаблона**: Возьмите за основу `assets/rule_template.md`.
3. **Анализ примеров**: Обратитесь к `references/rule_examples.md` для выбора подходящих формулировок и логики.
4. **Генерация файла**: Создайте новый файл в директории `.gemini/rules/` с именем `package_<name>.md`.

## Рекомендации по заполнению разделов

- **Соответствие имён**: Укажите все возможные синонимы корпуса, чтобы Gemini мог автоматически подбирать правило.
- **Footprint**: Опишите расположение выводов относительно друг друга в сетке отверстий.
- **bodyKeepout**: Четко определите, какие отверстия перекрываются корпусом (запрещены для других выводов).
- **Нумерация**: Опишите стандартную нумерацию для ориентации `rot=0` и уточните, что при повороте она сохраняется.

## Ресурсы скилла

- **`assets/rule_template.md`**: Базовый шаблон для копирования.
- **`references/rule_examples.md`**: Примеры реальных правил (DIP, AXIAL, TO-92) для понимания стиля.

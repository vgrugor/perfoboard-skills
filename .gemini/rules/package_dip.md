# Правила размещения: DIP (THT)

## Соответствие имён корпуса

Считать этим типом корпуса всё, что пользователь обозначает как:
- DIP-N, PDIP-N, DIL-N (где N — чётное число выводов)

## Геометрия perfboard (предпосылка)

- Сетка отверстий 2.54 мм, координаты целочисленные (x,y)
- Каждый вывод занимает ровно одно отверстие

## Footprint (по умолчанию)

- 2 параллельных ряда выводов
- Шаг выводов вдоль ряда: 1 отверстие (2.54 мм)
- Межрядовое расстояние: 3 отверстия (7.62 мм, “300 mil DIP”)

Если пользователь явно указывает другое межрядовое расстояние, использовать его вместо 3 отверстий.

## Нумерация выводов (по умолчанию)

При ориентации rot=0 (пины идут “вниз” по оси Y):
- Левый ряд: 1..(N/2) сверху вниз (y увеличивается)
- Правый ряд: N..(N/2+1) сверху вниз (то есть нумерация идёт против часовой стрелки вокруг корпуса)

При rot=90/180/270 поворачивать footprint и нумерацию как жёсткое тело.

Если пользователь задаёт иную нумерацию/ключ (точка/выемка), она имеет приоритет.

## Занимаемые отверстия

- pinHoles: N отверстий, по N/2 в каждом ряду
- bodyKeepout: прямоугольная зона, покрывающая:
  - все отверстия между рядами на длине корпуса
  - плюс 1 отверстие запаса по длине в обе стороны, если пользователь не запретил

Запрещено размещать чужие выводы в bodyKeepout (корпус перекрывает пайку/вставку). Допускается прохождение проводов снизу.

## Допуски и изгиб выводов

- Выводы DIP считаются “жёстко заданными” footprint’ом
- Изгиб выводов для изменения межрядового расстояния не допускается, если пользователь явно не разрешил
- Допускается только лёгкая подстройка по месту (не меняющая отверстия вставки)

## Рекомендации для оптимизации площади

- DIP по возможности ставить первым (крупный и задаёт “ядро” схемы)
- Питать DIP шинами рядом: один ряд/колонка под VCC, другой под GND (если есть такие неты)
- Рядом с DIP размещать детали, соединённые с соседними пинами, чтобы маршруты были короткими

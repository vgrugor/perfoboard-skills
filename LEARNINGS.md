# DIYLayout Project - Lessons Learned

Этот файл служит базой знаний для оптимизации процесса трассировки. Каждая запись должна предотвращать повторение прошлых ошибок.

## Паттерны Ошибок (Anti-Patterns)

1.  **"Центральный затор" (Center Congestion)**:
    *   *Ошибка*: Размещение всех компонентов (MCU, Транзисторы, Зуммеры) слишком близко друг к другу.
    *   *Последствие*: Автотрассировщик создает "лабиринты", которые блокируют шины питания и земли.
    *   *Решение*: Оставлять минимум 3-4 свободных отверстия между функциональными блоками.

2.  **"Смешанные слои без джамперов"**:
    *   *Ошибка*: Попытка развести пересекающиеся шины (VCC/GND) и сигналы на одном слое.
    *   *Последствие*: Неизбежные короткие замыкания (КЗ).
    *   *Решение*: Использовать стратегию "Силовых шин по краям" или явно вводить джамперы (Short Staple Jumpers) при первом же обнаружении пересечения.

3.  **"Неортогональные сегменты"**:
    *   *Ошибка*: Описание путей в JSON по диагонали или через недопустимые координаты.
    *   *Последствие*: Ошибки парсинга бенчмарка и "Heap out of memory".
    *   *Решение*: Строго придерживаться шага в 1 клетку и только по осям X/Y.

## Успешные стратегии (Best Practices)

1.  **"Шины по периметру"**: Разводка VCC по самому правому краю, а GND — между MCU и компонентами. Это освобождает центр для сигнальных линий.
2.  **"Горизонтальная ориентация TO-92"**: Разворот транзисторов выводами вдоль оси X (E-B-C) упрощает подвод сигналов к базе снизу или сверху, не перекрывая соседние ряды.
3.  **"Инкрементальная верификация"**: Запуск бенчмарка после каждой проложенной цепи, а не в самом конце.

# DIYLayout Project - Gemini CLI Standards

Этот проект предназначен для автоматизированного проектирования раскладок электронных компонентов на макетных платах (perfboards). В ходе работы мы выработали строгие стандарты качества и инструменты автоматизации.

## Архитектура проекта

- **`.gemini/skills/perfoboard-layout-optimizer/`**: Основной интеллектуальный модуль проекта. Содержит правила проектирования и инструменты трассировки.
- **`.gemini/rules/`**: Библиотека footprint-правил для различных корпусов (DIP, TO-220, AXIAL и др.).
- **`layout/`**: Директория для результатов работы. Здесь хранятся JSON и SVG файлы раскладок.

## Основные инструменты (внутри папки скилла)

1. **`tools/auto_route.js` (Smart Router)**: Алгоритмический трассировщик (A*), который автоматически прокладывает дорожки, обходя препятствия и устанавливая перемычки (jumpers) там, где это необходимо.
2. **`tools/score_layout.js` (Benchmark)**: Система оценки качества. Проверяет наличие КЗ, пересечений в узлах (Node Exclusivity), байпасов компонентов и рассчитывает итоговый Score.
3. **`tools/json2svg.js`**: Генератор визуализации для быстрого аудита.
4. **`viewer/index.html`**: Интерактивный веб-интерфейс для детального изучения раскладки.

## Инженерные стандарты (Mandates)

### 1. Безопасность и Валидация
- Любая раскладка считается завершенной только при **0 Short Circuits** в отчете бенчмарка.
- **Node Exclusivity**: Запрещено пересечение разных цепей в одном отверстии на нижнем слое.
- **Component Bypass**: Запрещено замыкание пассивных компонентов (R, C, LED) их собственными дорожками.

### 2. Топология и Эстетика
- **Short Staple Jumpers**: Перемычки должны иметь минимальную длину (принцип "короткой скобы"). Длинные "летающие" провода запрещены.
- **Bottom Layer Priority**: 90%+ трассировки должно быть выполнено на нижнем слое (пайка).
- **DIP-коридор**: К пинам микросхем трассы должны подходить перпендикулярно, чтобы не блокировать соседние отверстия.

### 3. Рабочий процесс (Workflow)
- Используй **Incremental Loop**: размещай одну деталь -> сразу выполняй её трассировку -> проверяй бенчмарком -> переходи к следующей.
- Всегда генерируй SVG файл после завершения работы для финального визуального аудита.

## Как запустить верификацию
```powershell
node .gemini/skills/perfoboard-layout-optimizer/tools/score_layout.js layout/your-file.json
```

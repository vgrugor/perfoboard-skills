# Примеры правил размещения компонентов

## Пример 1: DIP Корпус (Многовыводной, жесткий footprint)

```markdown
# Правила размещения: DIP (THT)

## Соответствие имён корпуса
Считать этим типом корпуса всё, что пользователь обозначает как:
- DIP-N, PDIP-N, DIL-N (где N — чётное число выводов)

## Footprint (по умолчанию)
- 2 параллельных ряда выводов
- Шаг выводов вдоль ряда: 1 отверстие (2.54 мм)
- Межрядовое расстояние: 3 отверстия (7.62 мм, “300 mil DIP”)

## Нумерация выводов (по умолчанию)
При ориентации rot=0 (пины идут “вниз” по оси Y):
- Левый ряд: 1..(N/2) сверху вниз (y увеличивается)
- Правый ряд: N..(N/2+1) сверху вниз (то есть нумерация идёт против часовой стрелки вокруг корпуса)
```

## Пример 2: AXIAL Корпус (2 вывода, гибкий footprint)

```markdown
# Правила размещения: AXIAL-2PIN (THT)

## Допустимые расстояния между отверстиями (lead span)
По умолчанию:
- minSpan: 2 отверстия
- preferredSpan: 4..7 отверстий
- maxSpan: 12 отверстий

## Изгиб выводов
Разрешены варианты:
- “прямой” монтаж: выводы без смещения (оба по одной линии)
- “собачья ножка” (dogleg): допустимо сместить один вывод на 1 отверстие перпендикулярно линии корпуса, если это снижает пересечения проводов
```

## Пример 3: TO-92 (3 вывода, треугольник или линия)

```markdown
# Правила размещения: TO-92 (THT)

## Footprint (по умолчанию)
- 3 вывода: pin1, pin2, pin3 (или “E/B/C” для транзистора)
- Шаг выводов: 1 отверстие (2.54 мм)
- Вариант 1 (Inline): Все 3 вывода в одну линию (например, по X)
- Вариант 2 (Triangle): 2 вывода в ряд, 3-й смещен на 1 отверстие перпендикулярно
```

# Perfoboard Layout Optimizer

## Назначение
Этот skill превращает описание схемы (детали + корпуса + соединения) в компактную раскладку на макетной плате, минимизируя площадь и гарантируя отсутствие электрических конфликтов.

## Когда использовать
- Пользователь дает размеры платы и список деталей (BOM).
- Нужно получить координаты каждой ножки, ориентацию элементов и план соединений (трассировку).

## Входные данные
1. **Размер платы**: width/height в отверстиях.
2. **Список компонентов (BOM)**: ref, package (согласно .trae/rules), pins.
3. **Нетлист**: словарь соединений вида netName -> ["ref.pin", ...].
4. **Предпочтения**: допускаются ли перемычки (джамперы), предпочтительные шины питания.

## Оптимизационная цель (Score)
Минимизировать (чем меньше, тем лучше):
1. **Валидность**: отсутствие наложений пинов и корпусов (Штраф: Infinity).
2. **Компактность (BBox Area)**: минимизация общей площади платы (Штраф: 5.0 за клетку).
3. **Форм-фактор (Perimeter & Aspect)**: 
    - Минимизация периметра (Штраф: 10.0 за единицу длины границы). Это предотвращает создание L-образных структур и "хвостов".
    - Соблюдение пропорций (Aspect Ratio < 2.5). Вытянутые платы штрафуются.
    - Мелкие элементы должны располагаться ПАРАЛЛЕЛЬНО длинной стороне крупных компонентов или в их "течении" (выше/ниже), не раздувая ширину.
4. **Короткие замыкания**: проход трассы через чужой пин или наложение ребер (Штраф: 1000 за каждое).
5. **Перемычки (Jumpers)**: число проводов на верхней стороне (Штраф: 50 за штуку).
6. **Проводимость (wireLength)**: (Штраф: 0.8 за сегмент).

## Правила и неизменяемые ограничения
- **Исключительность ребер**: Одно ребро сетки (нижний слой) = одна цепь.
- **Исключительность узлов (Node Exclusivity)**: Две разные цепи **не могут** пересекаться в одном отверстии (узле) на нижнем слое.
- **Приоритет нижнего слоя**: Основная длина трассы (минимум 90%) должна проходить по нижней стороне (пайка).
- **Минимализм перемычек (Staple Jumpers)**:
    - **Clean Via Rule**: Точки входа и выхода перемычки (via) **запрещено** совмещать с отверстиями, в которых установлены пины компонентов. 
    - Джампер должен начинаться в пустом отверстии, переходить на верхний слой, заканчиваться в другом пустом отверстии, переходить на нижний слой и только после этого соединяться с пином детали по нижней стороне.
    - Переход через отверстие, занятое пином той же цепи, на нижнем слое разрешен, но **не как точка смены слоя**.
    - Длина перемычки должна быть минимальной (идеально 2-3 клетки).

## Процесс (алгоритм)

### 0) Глобальная оптимизация размещения (Simulated Annealing)
- Использовать `node ./tools/anneal_place.js` для минимизации площади и приблизительной длины проводов.
- Скрипт переставляет компоненты и при необходимости очищает трассы.
- Пример запуска:
  `node ./tools/anneal_place.js layout/perfoboard-layout.json --iters 30000 --t0 10 --t1 0.1 --seed 42 --move 1`

### 5) Трассировка и использование перемычек
- **Стратегия "Short Hop"**:
    1. Вести трассу по нижнему слою до упора в препятствие (чужую дорожку).
    2. Установить точку перехода (via).
    3. Создать джампер минимальной длины (ортогональный или короткий прямой), чтобы перешагнуть препятствие.
    4. Вернуться на нижний слой сразу после препятствия.
- **Штрафы**: В функции оценки (Cost Function) штрафовать не только за количество перемычек, но и **за их длину**.

- **Component Bypass**: Цепь не может соединять два вывода одного пассивного компонента (R, C, D). Трасса должна разрываться на корпусе.
- **DIP-коридор**: К пинам микросхем подходить перпендикулярно, чтобы не блокировать доступ к соседним отверстиям ряда.
- **Двухслойность**: Основная разводка — снизу (пайка). Перемычки — сверху (изолированный провод).

## Процесс (алгоритм)

### 1) Инкрементальный цикл размещения (Incremental Loop)
Для сложных схем (например, NE555) рекомендуется использовать пошаговый метод:
1. **Установка "Якоря"**: Разместить **самый крупный** компонент (NodeMCU, ESP32, DIP-панель) в центр или на стратегическую позицию. Он служит основой, вокруг которой будет строиться остальная топология.
2. **Шаг детали**:
    - Выбрать следующую деталь из BOM (приоритет тем, что связаны с уже установленными).
    - Разместить её, соблюдая правила зазоров и гравитации.
    - **Немедленная трассировка**: Сразу провести все цепи этой детали к уже установленным компонентам с помощью `./tools/auto_route.js`.
    - **Валидация**: Запустить бенчмарк. Если Score вырос из-за КЗ — немедленно переставить деталь или изменить путь.
3. Повторять, пока все детали не будут установлены.

### 2) Трассировка (Routing)
- **Path Consolidation (Anti-Bubble)**: При трассировке (и использовании `auto_route.js`) всегда отдавать приоритет уже существующим сегментам той же цепи (стоимость 0.1 против 1.0 для новых). Это предотвращает "раздвоение" дорожек и создание параллельных путей (пузырей).
- **Segment Deduplication**: Никогда не добавлять в JSON сегмент, который уже существует (в любом направлении).
- **Occupancy Grid**: Вести виртуальную карту занятых отверстий и ребер.
- **Validated Tracing**: Для добавления сегментов использовать `./tools/trace_helper.js` (защита от КЗ).
- **Jumpers**: Если путь на нижнем слое заблокирован, использовать массив `jumpers` для перехода на верхний слой.

### 3) Верификация (Validation)
- **Benchmark**: Запуск `node ./tools/score_layout.js` для проверки Score и отсутствия КЗ.
- **Visual Audit**: Генерация `SVG` (`node ./tools/json2svg.js`) и проверка "глазами".
- **Interactive Viewer**: Для детального осмотра открыть файл `./viewer/index.html` в браузере и загрузить сгенерированный JSON.

## Инструментарий и CLI

Для работы с раскладкой используй следующие команды (пути относительно корня скилла):

1. **Автоматическая трассировка (`auto_route.js`)**:
   Добавляет оптимальный путь между двумя точками для указанной цепи в JSON-файл.
   ```bash
   node ./tools/auto_route.js <file.json> <netName> <x1> <y1> <x2> <y2>
   ```
   *Параметры*: `netName` — имя цепи, `x1,y1` — старт, `x2,y2` — финиш.

2. **Бенчмарк и проверка КЗ (`score_layout.js`)**:
   Рассчитывает итоговый Score и выводит список всех найденных конфликтов.
   ```bash
   node ./tools/score_layout.js <file.json>
   ```

3. **Генерация SVG (`json2svg.js`)**:
   Создает визуальное представление платы для быстрого аудита.
   ```bash
   node ./tools/json2svg.js <file.json>
   ```

4. **Оптимизация размещения (`anneal_place.js`)**:
   Использует алгоритм имитации отжига для минимизации площади и связей.
   ```bash
   node ./tools/anneal_place.js <file.json> --iters 30000 --t0 10 --t1 0.1 --seed 42 --move 1
   ```

## Формат результата (JSON)
Записывать в `./layout/perfoboard-layout.json` со структурой:
`{ board, bbox, components: [{ref, package, pins: [{name, x, y}]}], nets: [{name, segments: [{x1,y1,x2,y2}], jumpers: [...]}] }`

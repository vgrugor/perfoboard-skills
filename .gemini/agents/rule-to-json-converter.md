# Роль
Инженер-программист, специализирующийся на парсинге документации и генерации конфигурационных файлов для инструментов проектирования печатных плат (EDA).

# Задача
Преобразовать текстовые описания корпусов (footprints) из формата Markdown в структурированный JSON-формат, который понимают инструменты трассировки и размещения.

---

# Процедурный алгоритм

## Шаг 1 — Поиск исходных файлов
1. Выполнить поиск всех файлов `*.md` в директории `.gemini/rules/`.
2. Если список пуст, сообщить об ошибке и прекратить выполнение.

## Шаг 2 — Анализ и извлечение данных
Для каждого файла:
1. Прочитать содержимое через `read_file`.
2. Извлечь **ID корпуса** из заголовка `# Правила размещения: ID` или из раздела `## Соответствие имён корпуса`. Использовать наиболее краткое и общее имя (например, `DIP`, `BUZZER`, `TO-92`).
3. Найти описание **размеров корпуса** (bodyKeepout):
   - Если указано `квадрат размером NxN`, то `w = N`, `h = N`.
   - Если указан `прямоугольник от X1 до X2 и от Y1 до Y2`, то `w = (X2 - X1 + 1)`, `h = (Y2 - Y1 + 1)`.
   - Для DIP-корпусов рассчитать `h` как `(количество пинов / 2) + 2` (с учётом запаса), `w` как `межрядовое расстояние + 1`.
   - Если размеры не указаны явно, использовать здравый смысл исходя из стандартных размеров компонентов на сетке 2.54 мм.
4. Установить значение `clearance` (по умолчанию `1`, если не указано иное в правиле).

## Шаг 3 — Формирование и сохранение JSON
1. Сформировать итоговый JSON-объект. Пример структуры:
```json
{
  "packages": {
    "DIP8": { "w": 4, "h": 6 },
    "BUZZER": { "w": 5, "h": 5 },
    "TO-92": { "w": 3, "h": 3 }
  },
  "clearance": 1
}
```
2. Сохранить результат в файл `rules/packages.json`.
3. Если пользователь требует отдельные файлы для каждого корпуса, создавать файлы вида `rules/[package_id].json` с аналогичной структурой (в объекте `packages` будет один ключ).

# Инженерные стандарты
- Все размеры (`w`, `h`) должны быть **целыми числами**, кратными шагу сетки (holes).
- Имена корпусов должны быть в **ВЕРХНЕМ РЕГИСТРЕ** для единообразия, если в нетлисте не используется иной стандарт.
- Если в правиле .md указано `hasClearanceUnderneath: true`, это свойство не обязательно переносить в `packages.json`, если инструмент его не требует, но размеры `w` и `h` должны отражать зону отчуждения (bodyKeepout).
- По умолчанию `clearance` всегда равен `1`.

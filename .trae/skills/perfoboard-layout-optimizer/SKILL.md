## Правила и неизменяемые ограничения

- **Исключительность ребер**: Одно ребро = одна цепь.
- **Исключительность отверстий (Hole Conflict)**: Трасса не может проходить через отверстие с чужим пином.
- **Запрет байпаса (Component Bypass)**: Одна цепь (Net) НЕ МОЖЕТ соединять два разных вывода одного и того же пассивного компонента (резистора, конденсатора, диода), если это не предусмотрено схемой. Трасса должна прерываться на компоненте: вход -> пин1 | корпус | пин2 -> выход.
- **Разрыв корпуса**: Для осевых деталей (AXIAL) запрещено проводить трассу той же цепи «под корпусом» от пина к пину, создавая короткое замыкание самого компонента.

## Процесс (алгоритм)

### 5) Трассировка и проверка на "сквозные" ошибки
- **Виртуальная сетка занятости (Occupancy Grid)**: Перед генерацией сегментов составить в памяти таблицу всех занятых отверстий (пинами) и всех занятых ребер (сегментами).
- **Валидированная трассировка**: Для добавления каждого сегмента в JSON использовать вспомогательный скрипт (например, `node scripts/trace_helper.js`). Это предотвратит:
    - Проход трассы через отверстие, занятое пином другой цепи (**Hole Conflict**).
    - Наложение трасс разных цепей на одно ребро (**Edge Conflict**).
- **Контроль байпаса**: Убедиться, что сегменты разных нетов прерываются на выводах пассивных компонентов (R, C, LED).
- **Двойной контроль**: После завершения трассировки всех цепей прогнать финальный файл через бенчмарк (скрипт `score_layout.js`).
